Build another toolkit for OG:image previewing and site metadata inspection called **"Previewer Toolkit"**.

The toolkit must be a production-ready, highly optimized, URL-driven image generation and metadata inspection system. It must function reliably on shared hosting environments and be designed for real-world usage where generated image URLs will be embedded directly inside HTML `<head>` tags as Open Graph images, Twitter cards, Discord embeds, LinkedIn previews, Telegram previews, and other social sharing systems.

The toolkit must be completely stateless and should not require user accounts, saved projects, template storage, user databases, queues, background workers, or scheduled jobs.

The system must generate images entirely from URL parameters and return the requested image immediately.

Example:

`https://awantools.site/plugins/preview/render?category=og&template=github_dark&heading=Developer%20Toolkit&description=200+%20Developer%20Tools&icon=code&format=png`

The response must be a valid image file and not an HTML page.

The image endpoint must be safe for use directly inside:

```html
<meta property="og:image" content="https://awantools.site/plugins/preview/render?...">
<meta name="twitter:image" content="https://awantools.site/plugins/preview/render?...">
```

The toolkit must be designed around two major sections:

1. Preview Image Generator
2. Site Metadata Inspector

The engine must:

* Validate all parameters
* Sanitize all inputs
* Apply defaults
* Prevent malformed requests
* Handle missing values gracefully

RELIABILITY REQUIREMENTS

* Never return partial images

* Return fallback images on failure
